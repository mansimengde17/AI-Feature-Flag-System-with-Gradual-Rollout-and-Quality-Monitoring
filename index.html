Feature flags that understand quality

AI Feature Flag System with Gradual Rollout

Feature flags built for AI features, where working is a quality gradient and not a boolean. Flags roll out in stages, quality is scored against the baseline continuously, and a bad...

Why boolean flags fall short

A traditional flag tells you the new code path did not crash. It cannot tell you the new prompt quietly made summaries worse for a fifth of your users. AI features fail on a gradient, so the flag has to measure quality, not just on or off.

Why it lands interviews

It proves you understand that shipping AI features needs different operational patterns than shipping normal code, and that you can build the tooling to support that difference.

What it protects against

A degraded prompt reaching everyone. Rollout is staged, each stage is gated on quality, and a sustained drop in the worst decile triggers an automatic rollback with an alert.

Why the numbers are trustworthy

Users are assigned by a consistent hash, so raising the percentage only adds users and never reshuffles them, and a canary t test blocks any advance that is statistically worse than baseline.

01
Request in

An incoming request carries a user ID that drives flag evaluation.

02
Hash assign

User ID is hashed to a stable cohort; raising the percentage only adds users, never reshuffles.

03
Flag evaluate

System checks the current rollout percentage and routes the request to the correct variant.

04
Score quality

A judge model scores the variant response against the baseline on every request.

05
Canary check

A statistical test gates each stage advance; it blocks any variant that is significantly worse.

06
Advance or rollback

Good variant advances through 1, 5, 25, 50, 100 percent; bad variant auto rolls back.

1
Consistent hash ring

Deterministic cohort assignment ensures reruns produce the same user assignments.

2
Flag evaluation API

Fast path with flag state cached in Redis; cold path reads from the config store.

3
Shadow execution engine

Runs both variants on every request and collects quality scores without user delay.

4
Judge model scorer

Scores each variant response against a rubric and records the result per user.

5
Staged rollout controller

Advances through configured percent gates after each canary check passes.

6
Rollback trigger

Monitors a rolling quality window; fires a Slack alert and resets rollout on sustained drop.
